1. 单文件组件
    前面使用Vue.component注册全局组件，仅仅是将Vue当成一个Js库来使用，而不是当成一个Js前端框架
    而且，这种全局注册组件的方式还存在以下几个问题:
        1) 组件名不能重复
        2) 缺乏语法高亮及语法提示功能
        3) 不支持CSS
        4) 没有构建步骤
    为了解决这些问题，Vue推出了单文件组件，文件扩展名 .vue
    它必须使用webpack构建工具才能使其工作

    一个Vue文件通常包含三部分:
        1) tempplate,视图模板,模板中必须包含一个根节点
        2) script,脚本逻辑
        3) style,样式,默认是全局的,加个scoped属性就变成了当前组件有效

2. vue-cli
    是官方提供的一个快速构建单页面应用的脚手架工具，自动生成Vue项目的目录
    并且集成了前端中的其他工具:babel,eslint,webpack...


    安装
        cnpm i -g @vue/cli
        npm install -g @vue/cli
    检查是否安装成功
        vue -V查看安装的版本号
    初始化Vue项目
        方式一: vue create 项目名称
        方式二: vue ui

    项目创建成功后，切换进项目根目录，使用命令启动:
        npm run serve

    vue-cli生成项目的目录结构:
        /--
            |-- node_modules
            |-- public
            |-- src
                |-- assets/ 静态资源文件
                |-- components/ 存放组件
                |-- App.vue 根组件
                |-- main.js 启动文件
            |-- .gitignore
            |-- babel.config.js
            |-- package.json

3. 路由
    1) 前端的路由是指SPA的路径的管理器
        vue-router插件是官方提供的一个与vue.js深度集成的插件
        Vue的单页面应用(SPA)基于组件和路由
        路由的本质是建立url和组件之间的映射关系
        在开发SPA时，只能使用一个页面，所以不能使用a链接去跳转页面，所以只能使用路由来管理页面之间的切换
    2) 路由的安装
        手动安装：vue add router
        使用脚手架：

    3) 路由的配置文件 router/index.js

    4) 导航链接
        <router-link to="/hello">Hello</router-link>

        to="/hello" 表示跳转的路由路径

    5) 在根目录下配置二级页面
        在路由的index.js中配置二级页面
        routes:[
            {
                path:'路由的路径',
                name:'路由名称',
                component:路由组件
            }
        ]
        当路由触发时，会在App.vue指定的位置(router-view标签)显示对应路由组件的内容
        路由的触发：通过router-link导航链接

    6) 路由的命名
        有时候需要展示多个视图，就需要多个渲染出口(router-view标签) 设置name属性

    7) 路由的嵌套(子路由)
        配置路由时在父路由中设置children:[]
        配置子路由 
        {
        path: '子路由路径', 不需要加/
        component: 子路由组件
        }
        父路由中设置导航链接
        <router-link to="/父路由路径/子路由路径"></router-link>
        同时在父组件中设置router-view
